# Supportability Value Toolkit

**A program-management framework, working automation, and an ROI model that turn product supportability into measurable dollars — on both the cost-saving and revenue sides.**

> Built by a Senior Supportability Program Manager to show how disciplined supportability practice reduces support cost *and* protects/grows revenue. All data here is synthetic — no confidential or customer information.

---

## Why this repo exists

Supportability is usually pitched as a cost centre. It isn't. Done well it does two things at once:

- **Saves cost** — fewer cases, faster resolution, less escalation, less engineering interrupt.
- **Protects and grows revenue** — higher renewal/retention, faster time-to-value, and more room for upsell/attach because customers actually succeed with the product.

This repo makes that case concretely: a written framework, runnable tools, and a spreadsheet model you can point at your own (sanitised) numbers.

## The 30-second business case

A mid-size commercial support operation handling **10,000 cases/year** at **AUD 85 per case**:

| Lever | Mechanism | Modeled annual impact |
|-------|-----------|-----------------------|
| Case deflection (self-service + KB) | 18% of cases never opened | **AUD 153,000 saved** |
| Escalation reduction | 30% fewer engineering escalations | **AUD 96,000 saved** |
| Faster resolution | 15% lower handle time | **AUD 103,000 saved** |
| Retention lift | +1.5pt renewal on supported accounts | **AUD 420,000 protected revenue** |
| Time-to-value / attach | Faster onboarding enables upsell | **AUD 60,000 revenue enabled** |

**Total modeled annual value: AUD 832,000** — AUD 352,000 cost saving + AUD 480,000 revenue impact.

> Numbers are illustrative and generated by [`roi-model/`](roi-model/). Swap in your own inputs — every figure is a formula, not a hard-code.

## Quick start

\`\`\`bash
# 1. Analyse the top support drivers in the sample case data
python tools/python/support_driver_analyzer.py

# 2. Build the ROI model spreadsheet from the inputs file
python roi-model/build_roi_model.py
# -> writes roi-model/supportability_roi_model.xlsx
\`\`\`

PowerShell users:

\`\`\`powershell
# Find knowledge-base gaps: high-volume drivers with no KB article
pwsh tools/powershell/Get-KBGapReport.ps1
\`\`\`

## Design principles

1. **Every claim is falsifiable.** Metrics are defined so anyone can reproduce them.
2. **Cost and revenue, not just cost.** Supportability is scored on both sides of the ledger.
3. **Runs out of the box.** Sample data ships with the tools; no setup, no secrets.
4. **Portable, not proprietary.** The framework maps to any commercial support org.
